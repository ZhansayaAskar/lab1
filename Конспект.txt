Конспект по Git


Что такое Git?
Git — система контроля версий. Позволяет отслеживать изменения в проекте, 
возвращаться к предыдущим версиям, работать в команде.

"Репозиторий" — хранилище проекта и его истории (.git — скрытая папка с данными).
"Коммит" — «снимок» изменений в проекте. Коммиты образуют историю.
"Ветка" — параллельная версия проекта. Обычно основная ветка называется main.

Создание проекта

mkdir work - Создание новой папки с именем work

cd work - сменить папку

touch hello.html - команда для создания пустого файла 

git init - Инициализация Git-репозитория

git add hello.html - добавление всех новых/изменённых файлов в текущей папке.

git commit -m "Initial commit" - commit = сохранение «снимка» текущего состояния файлов.
                                 -m "..." — короткое сообщение к коммиту (обязательно, чтобы понимать историю )

Проверка состояния
git status - Показывает, что изменено, что готово к коммиту

История изменений
git log              # полная история
git log --oneline    # краткий вид
git log --author="Имя"
git log --since="7 days ago"

Возврат к старым версиям

git log --oneline - Посмотреть хеш коммита

git checkout <hash> - Переключиться на коммит

git switch main - Вернуться в основную ветку

Теги

git tag v1 - Создать тег
git tag - Список тегов
git checkout v1 - Переключение по тегу
git checkout v1^     # один шаг назад  - Указание на предыдущий коммит
git checkout v1~1


Отмена изменений
Если изменения ещё не добавлены в индекс - git restore hello.html
Если изменения уже проиндексированы - git restore --staged hello.html
Команда restore с флагом --staged очищает индекс
